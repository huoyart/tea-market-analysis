# 启动说明

## 问题排查

如果遇到 `ERR_CONNECTION_REFUSED` 错误，请按以下步骤操作：

### 1. 检查依赖是否已安装

在项目目录下，检查是否存在 `node_modules` 文件夹。如果不存在，需要先安装依赖：

```bash
npm install
```

### 2. 手动启动开发服务器

安装完依赖后，运行：

```bash
npm run dev
```

### 3. 检查端口占用

如果 3000 端口被占用，Vite 会自动尝试其他端口。请查看终端输出的实际端口号。

### 4. 使用批处理文件启动

直接双击 `start.bat` 文件，它会自动：
- 检查并安装依赖
- 启动开发服务器

### 5. 如果仍然无法访问

1. 检查防火墙设置
2. 尝试使用 `http://127.0.0.1:3000` 而不是 `localhost:3000`
3. 查看终端是否有错误信息

## 常见问题

- **端口被占用**：修改 `vite.config.js` 中的端口号
- **依赖安装失败**：检查网络连接，或使用 `npm install --registry https://registry.npmmirror.com`
- **Node.js 版本**：确保 Node.js 版本 >= 14
